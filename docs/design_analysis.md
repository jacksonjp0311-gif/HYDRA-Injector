# Design Analysis

## Deep Extraction

The HYDRA v1.4 gist defines a canonical residual operator:

```text
H = Seal o Retract o Inject o Anchor
```

The document's core theory is not merely "inject something." It is governed injection:

1. Anchor an admissible masked domain.
2. Inject a normalized precursor field.
3. Retract controlled bulk amplitude.
4. Seal the residual through constrained smoothing.
5. Report boundedness, curvature, omega, mass, and nontriviality.
6. Test robustness through bounded perturbation.
7. Preserve artifact type, claim boundary, and non-claim locks.

## Cross-Domain Translation

HYDRA Injector translates the theory into a portable field operator. The cross-domain grounding is:

- projection methods: admissibility constraints and projection discipline,
- image inpainting: masked-domain smoothing with boundary conditions,
- diffusion segmentation: local constrained propagation,
- distance transforms: boundary-band reasoning,
- artifact governance: header discipline to prevent claim drift.

## What Is New Here

The repository combines the residual operator with governance:

```text
numerical operation + perturbation basin + archive gate
```

That means the tool can say more than "the output is smooth." It can report whether the run stayed bounded, nontrivial, admissible, and properly typed.

## Code Injection Layer

The codeweave layer maps HYDRA's operator into source-code edits:

- anchor: explicit repository root, target file, and marker,
- inject: agent-authored code block,
- retract: path, extension, size, and forbidden-pattern checks,
- seal: unified diff or explicit apply result.

This makes agent code injection reviewable and bounded. The tool does not execute injected code.

## Safety Boundary

Despite the name, this is not a prompt-injection or exploit tool. It is a controlled scalar-field injection harness for defensive, scientific, and governance-oriented use.
